# ADR 0001 - Journal first, brique par brique

## Contexte

Le projet global est large (portfolio, RAG, Obsidian, MCP).  
Sans ordre strict, le risque est la dispersion et une historique GitHub difficile a lire.

## Decision

Adopter un enchainement par briques:

1. AI-Lab-Journal (vitrine publique)
2. rag-core (brique RAG)
3. obsidian-vault (interconnexion native)
4. briques suivantes selon priorites

## Consequences

- Avancement plus lisible pour humains et IA.
- Historique GitHub plus propre.
- Moins de charge mentale lors des transitions de contexte.
